File Operation Flags


This page contains a list of the flag constants that can be used with these file operation functions: _FileCopyDlg, _FileMoveDlg, _FileRenameDlg, and _FileDeleteDlg.

Note: The "Value" is given just for reference, you should always use the "name" of the flag in order to keep your scripts readable.


Flag Value Description
   

All flag constants are defined within the library. No additional includes required.
Default/forced for all operations. 0x0000 Use default parameters.
$FOF_ALLOWUNDO 0x0040 Copies the files specified in pFrom ($source) to the location specified in pTo ($dest).
$FOF_CONFIRMMOUSE 0x0002 Not currently implemented.
$FOF_FILESONLY 0x0080 Perform the operation on files only if a wildcard file name (*.*) is specified.
$FOF_MULTIDESTFILES 0x0001 The pTo ($dest) member specifies multiple destination files (one for each source file) rather than one directory where all source files are to be deposited.
$FOF_NOCONFIRMATION 0x0010 Respond with "Yes to All" for any dialog box that is displayed.
$FOF_NOCONFIRMMKDIR 0x0200 Does not confirm the creation of a new directory if the operation requires one to be created.
$FOF_NOCOPYSECURITYATTRIBS 0x0800 Do not copy NT file Security Attributes.
$FOF_NOERRORUI 0x0400 No user interface will be displayed if an error occurs.
$FOF_NORECURSION 0x1000 Do not recurse directories (i.e. no recursion into subdirectories).
$FOF_RENAMEONCOLLISION 0x0008 Give the file being operated on a new name in a move, copy, or rename operation if a file with the target name already exisits.
$FOF_SILENT 0x0004 Does not display a progress dialog box.
$FOF_SIMPLEPROGRESS 0x0100 Displays a progress box but does not show the file names.
$FOF_WANTMAPPINGHANDLE 0x0020 If $FOF_RENAMEONCOLLISION is specified, the hNameMappings member will be filled in if any files were renamed.
$FOF_NO_CONNECTED_ELEMENTS 0x2000 Do not operate on connected elements. Requires Internet Explorer 5 or higher.
$FOF_WANTNUKEWARNING 0x4000 During delete operations, warn if permanent deleting instead of placing in recycle bin (partially overrides $FOF_NOCONFIRMATION). Requires Internet Explorer 5 or higher.
$FOF_NORECURSEREPARSE 0x8000 Currently undocumented. Not recommended.
